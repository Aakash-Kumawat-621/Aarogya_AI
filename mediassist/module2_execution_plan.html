High effort ~3 hrs
Questions you must answer before writing any code
Write answers to these on paper. They become your spec for Gemini in S3:
  1. What entity types do you want extracted?
    Minimum: SYMPTOM, BODY_PART, DURATION, SEVERITY, DISEASE_HISTORY, NEGATION_FLAG
    Decision: do you want MEDICATION extracted here too, or leave that for Textract OCR?

  2. How do you handle negation?
    "no chest pain" must NOT be treated as "chest pain".
    "denies fever" = negated fever.
    Decision: use spaCy's dependency parser to detect negation tokens (no, not, denies, without, never) within 3 tokens of an entity.

  3. How do you handle uncertainty?
    "possible pneumonia", "might have diabetes" = uncertain.
    Decision: flag entities with uncertainty_flag=True when preceded by: possible, possible, maybe, might, could, suspected.

  4. How do you handle duration?
    "chest pain for 3 days" → extract duration="3 days", map to duration_category: acute (<7 days), subacute (7–30 days), chronic (>30 days).

  5. What is the output schema? Design this yourself:
# Output schema — decide this yourself
{
  "extracted_symptoms": [
    {
      "text": "chest pain",
      "canonical_form": "chest pain",  # after normalization
      "body_part": "chest",
      "negated": false,
      "uncertain": false,
      "severity": "severe",   # mild/moderate/severe/None
      "duration": "3 days",
      "duration_category": "acute"
    }
  ],
  "disease_history_mentions": ["diabetes", "hypertension"],
  "raw_text": "original input",
  "confidence": 0.85
}
Why this is yours and not Gemini's
If you let Gemini design the NLP output schema, it will produce something generic — probably just a list of entity strings. Your schema above is what separates a portfolio-worthy medical NLP system from a toy. The negation handling, uncertainty flags, and duration categorization are medically informed decisions that an interviewer will ask you to defend.

Design the PatientContext schema and merge rules
(thinking task — no code yet)
The most architecturally important task in the entire project. PatientContext is the object that every downstream module reads. Its schema and merge rules determine the quality of diagnosis, urgency scoring, and RAG retrieval. Design it on paper first.
High effort ~4 hrs
Answer these questions on paper — they become your merge rules
  1. What happens when inputs conflict?
    Example: patient says "mild cough" (text), X-ray shows "possible pneumonia 78% confidence" (image).
    Rule decision: X-ray finding takes priority over subjective symptom description for severity assessment, but both are passed downstream. The RAG query includes both. The LLM reconciles.
  2. What happens when an input is missing?
    Not every patient uploads everything. Define: if no X-ray → xray_findings = None. If no prescription → prescription_data = None. None values must be gracefully handled everywhere downstream without errors.
  3. How does the personal profile attach as risk modifiers?
    Design the computed risk_flags field: a list of strings that summarize the patient's risk profile for easy injection into the RAG prompt.
    Example: age=55, smoker, hypertension → risk_flags = ["cardiac_risk_high", "respiratory_risk_high", "age_over_50"]
  4. What is the overall confidence of the context?
    If only symptoms text is provided → context_confidence = 0.5
    If symptoms + X-ray → context_confidence = 0.75
    If all 5 inputs → context_confidence = 0.95
    This tells the LLM how much evidence it has to work with.
Design the PatientContext schema yourself

# backend/app/api/routes/analyze.py — update yourself
from app.core.patient_context import build_patient_context
from app.models.request_models import AnalyzeRequest, PatientProfile
from app.models.response_models import AnalyzeResponse

@router.post("/analyze")
async def analyze(
    symptoms_text: Optional[str] = Form(None),
    patient: str = Form(...),       # JSON string
    location: Optional[str] = Form(None),
    xray_image: Optional[UploadFile] = File(None),
    body_photo: Optional[UploadFile] = File(None),
    prescription: Optional[UploadFile] = File(None),
):
    # 1. Parse patient profile
    patient_profile = PatientProfile.model_validate_json(patient)

    # 2. Read image bytes if provided
    xray_bytes = await xray_image.read() if xray_image else None
    photo_bytes = await body_photo.read() if body_photo else None
    rx_bytes = await prescription.read() if prescription else None

    # 3. Build patient context
    context = await build_patient_context(
        symptoms_text=symptoms_text,
        patient_profile=patient_profile,
        xray_image_bytes=xray_bytes,
        body_photo_bytes=photo_bytes,
        prescription_bytes=rx_bytes,
        location=LocationData.model_validate_json(location) if location else None
    )

    # 4. For now — return context as partial response
    # Module 3 will add RAG chain here
    return {
        "session_id": context.session_id,
        "context_built": True,
        "inputs_processed": context.inputs_provided,
        "risk_flags": context.risk_flags,
        "symptoms_extracted": len(context.symptom_entities),
        "context_confidence": context.context_confidence,
        "status": "context_ready_rag_pending"  # placeholder until Module 3
    }
