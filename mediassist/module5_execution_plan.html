Implement finder.py — Google Places Nearby Search
backend/app/modules/doctors/finder.py
Gemini implements the Google Places API call with your specialty_mapper output. Takes specialist search term + user lat/lng + radius, returns top-5 doctors ranked by rating × inverse distance. Includes filtering, deduplication, and distance calculation.
Medium ~2 hrs
Prompt to give Gemini
Implement backend/app/modules/doctors/finder.py for MediAssist AI. Main function: async def find_doctors( condition: str, patient_profile: PatientProfile, location: LocationData, urgency_level: str, radius_km: int = 15 ) -> list[DoctorResult] Steps: 1. Call specialty_mapper.get_specialty(condition, patient_profile) to get search term 2. If urgency_level == "EMERGENCY" and specialty_mapper says emergency_dept=True: - Search for "emergency hospital" instead of specialist - Set radius_km = 5 (closest emergency, not best rated) 3. Call Google Places Nearby Search API: URL: https://maps.googleapis.com/maps/api/place/nearbysearch/json Params: location={lat},{lng}, radius={radius_km*1000}, keyword={search_term}, type=doctor|hospital Key: settings.GOOGLE_PLACES_API_KEY 4. For each result, call Places Details API to get: - phone_number, opening_hours, website, rating, user_ratings_total 5. Calculate distance from user location using geopy.distance.geodesic 6. Rank by: composite_score = (rating/5 * 0.6) + ((1 - dist_km/radius_km) * 0.4) (Higher rating AND closer = better score) 7. Return top-5 as DoctorResult objects DoctorResult fields: name, specialty, hospital, address, phone, rating, distance_km, google_place_id, is_open_now Handle: API quota exceeded → fall back to NMC registry (S3) Handle: No results within radius → expand to radius_km*2 and retry once Handle: missing phone/rating → use None, don't crash Cache results per (condition, lat, lng) in DynamoDB with 24hr TTL (avoid repeat API calls)
Google Places API cost control
Google Places gives $200 free credit monthly. Nearby Search = $0.032/call. Details = $0.017/call. Per analyze request with 5 results: ~$0.032 + (5 × $0.017) = $0.117. At $200 credit: ~1,700 free requests/month.

The DynamoDB cache is critical — repeated searches for "cardiologist in Jaipur" return cached results instead of calling the API again. Cache key: f"{condition}_{lat:.2f}_{lng:.2f}" (rounded to 2 decimal places ≈ 1.1km precision).

Common issues to look for
  • Medical stores appearing as doctors — Google Places sometimes returns pharmacies for "doctor" queries. Fix: add exclude_types=["pharmacy","drugstore"] to Places query.
  • Wrong city results — If radius is too large, results from Delhi appear. Keep radius ≤ 25km for Jaipur.
  • Permanently closed — Google Places API returns business_status: "CLOSED_PERMANENTLY". Filter these out.
  • No phone number — Many Indian clinics don't have phone numbers in Google. Show "Contact via Google Maps" as fallback.

def get_disclaimer(urgency_level: str) -> str:

Think about each disclaimer carefully:
