You cannot index all of PubMed (35M papers). You must decide what disease scope your system covers, which PubMed MeSH terms to query, what OpenFDA fields to include, and how to ensure quality over quantity. This shapes everything downstream.
High effort ~4 hrs
Decisions you must make before S2
  1. What disease categories does MediAssist AI cover?
    You cannot cover everything. Define your scope explicitly. Recommended for MVP:
    • Respiratory: pneumonia, asthma, COPD, tuberculosis, COVID-19
    • Cardiovascular: chest pain, hypertension, cardiac arrhythmia, heart failure
    • Gastrointestinal: abdominal pain, appendicitis, gastritis, IBS
    • Musculoskeletal: fractures, sprains, joint pain, back pain
    • Dermatological: rashes, infections, burns, wounds
    • Neurological: headache, migraine, dizziness, seizures
    • Infectious: dengue, malaria, typhoid, common infections (India-relevant)
    • Endocrine: diabetes symptoms, thyroid disorders
    Write this list down. It becomes your PubMed query MeSH terms.
  2. Chunk size decision — 256 vs 512 tokens?
    512 tokens: more context per chunk, better for complex medical explanations. Fewer total chunks.
    256 tokens: more precise retrieval, better for specific symptom–condition pairs.
    Your decision: 512 tokens with 50-token overlap for medical-kb (explanations need context), 256 tokens for drug-db (drug facts are short and specific).
  3. How many chunks to index?
    Pinecone free tier: 100K vectors max. Budget: medical-kb=60K, drug-db=30K, hospital-db=10K.
    At 512 tokens/chunk: 60K chunks ≈ 30M tokens ≈ roughly 4,000–6,000 PubMed abstracts.
  4. Quality filter for PubMed?
    Only index: abstracts published after 2018, English language, human studies only (filter out animal studies), abstract length > 150 words.
    This filters out low-quality and irrelevant papers.

For each chunk you read, ask yourself:
  • Is it useful? Does it contain clinical information (symptoms, diagnosis, treatment)?
  • Is it complete? Does it make sense as a standalone passage, or is it cut off mid-sentence?
  • Is it clean? No reference lists, no citation numbers, no HTML artifacts?

Common quality issues to look for
  • Reference sections being chunked — "1. Smith J, et al. 2019. JAMA 45:123" — useless. Fix: strip text after "References" or "Bibliography" before chunking.
  • Mid-sentence cuts — chunk ends with "patients with chest pain should be evalu-". Fix: increase chunk overlap to 100 tokens.
  • Metadata-only chunks — "Received: 2019-03-15. Accepted: 2019-04-01. DOI: 10.1016/..." — useless. Fix: filter chunks under 100 characters.
  • Wrong disease category — chunk about cardiac surgery in "dermatology" namespace. Fix: review your MeSH query terms.

Expected finding — what you will likely see
Dense retrieval wins on common conditions (chest pain, fever, cough) — BioSentBERT captures semantic meaning well for well-represented conditions.

Hybrid wins on rare diseases (Lemierre, Kounis) — exact term matching in BM25 catches what embeddings miss because rare conditions are underrepresented in the training data.

Decision you will likely make: Use hybrid search. Implement BM25 as a fallback only — try dense first, if top result score < 0.65 then rerank with hybrid. This avoids the overhead of BM25 for common queries while covering rare diseases.

# Function 1: builds the Pinecone retrieval query string
def build_retrieval_query(context: PatientContext) -> str:
    parts = []

    # Add confirmed (non-negated) symptoms
    confirmed = [e.canonical_form for e in context.symptom_entities if not e.negated]
    if confirmed:
        parts.append(" ".join(confirmed))

    # Add patient demographics for context
    p = context.patient_profile
    parts.append(f"{p.age}-year-old {p.gender}")

    # Add existing conditions
    if p.conditions:
        parts.append("history of " + ", ".join(p.conditions))

    # Add X-ray finding if present
    if context.xray_findings and context.xray_findings.top_condition:
        parts.append(f"X-ray finding: {context.xray_findings.top_condition}")

    # Add body photo finding if present
    if context.body_photo_findings:
        findings = [f["finding"] for f in context.body_photo_findings.get("findings", [])]
        if findings:
            parts.append("visible: " + ", ".join(findings))

    return " ".join(parts)

# Function 2: builds the patient_context_summary for LLM prompt
def build_context_summary(context: PatientContext) -> str:
    # Structured summary injected into the Bedrock prompt
    p = context.patient_profile
    summary = f"""
Patient: {p.name}, {p.age}yo {p.gender}, BMI: {bmi(p)}
Conditions: {', '.join(p.conditions) or 'none reported'}
Allergies: {', '.join(p.allergies) or 'none reported'}
Medications: {', '.join(p.medications) or 'none reported'}
Smoking: {p.smoking.value}{f', {p.pack_years} pack-years' if p.pack_years else ''}
Risk flags: {', '.join(context.risk_flags) or 'none'}

Reported symptoms (confirmed): {', '.join([e.canonical_form for e in context.symptom_entities if not e.negated])}
Denied symptoms: {', '.join([e.canonical_form for e in context.symptom_entities if e.negated])}
Disease history mentions: {', '.join(context.disease_history_mentions) or 'none'}
"""
    if context.xray_findings:
        summary += f"\nX-ray: {context.xray_findings}"
    if context.prescription_data:
        summary += f"\nPrescription drugs: {[m['name'] for m in context.prescription_data.get('medicines', [])]}"
    return summary.strip()
