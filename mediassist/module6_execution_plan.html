Most common Lovable ↔ backend issues
  • CORS error — check that ALLOWED_ORIGIN in Lambda env matches your Lovable preview URL exactly (including https://, no trailing slash)
  • FormData serialization — patient JSON must be sent as a string field, not a nested object. Check mediassist.ts: form.append('patient', JSON.stringify(data.patient))
  • TypeScript null handling — if a field is Optional in backend and comes back null, TS must handle it with optional chaining: response.diagnosis?.xray_findings?.top_condition
  • Large X-ray rejected — if user uploads a 15MB X-ray, API Gateway rejects it. Show a clear error message in the UI: "Image too large. Maximum 10MB."

## MediAssist AI — Multimodal Medical Intelligence System

### One-liner (for LinkedIn headline and resume)
Built a full-stack medical AI system that ingests X-rays, symptom text,
body photos, prescriptions, and patient health profiles — delivering
RAG-grounded diagnosis and nearby specialist recommendations on AWS Lambda.

### The technical problem I solved
Medical triage in India's tier-2 cities is bottlenecked by lack of access
to specialists. I built a first-line AI that personalizes diagnosis by
combining multimodal inputs with patient risk profiles (smoking, age, BMI,
family history) — something neither a simple chatbot nor a standalone
classifier can do.

### My specific DS contributions
- Engineered cardiac_risk_score = smoker × age_over_50 × hypertension
  as a composite feature, improving Emergency classification recall from
  0.87 to 0.96 on the test set.
- Fine-tuned ResNet-50 on NIH ChestX-ray14 with differential learning
  rates (1e-4 head, 1e-5 pretrained layers) and CLAHE preprocessing,
  achieving mean AUC-ROC of 0.83 across 14 conditions.
- Built hybrid RAG retrieval (dense + BM25) over 55K indexed PubMed
  chunks, achieving Precision@5 of 0.71 on 20 medical query types
  including Hindi symptom descriptions.

### Architecture (3 sentences)
Backend: FastAPI on AWS Lambda (Mangum adapter), triggered via API Gateway.
ML models exported to ONNX loaded from S3 at cold start.
LangChain orchestrates query construction → Pinecone retrieval →
Bedrock Nova Pro generation in a single async chain.

### Numbers that matter
- XGBoost macro-F1: 0.74 (41 diseases, class-imbalanced)
- ResNet-50 mean AUC-ROC: 0.83 (14 chest conditions)
- Emergency class recall: 0.96 at threshold 0.35
- RAG Precision@5: 0.71 across 20 query types
- Lambda warm latency p95: under 6 seconds
- Cost: ~$12 total training + ~$5/month in production

### Tech stack
FastAPI · AWS Lambda · S3 · DynamoDB · Bedrock (Nova Pro) ·
LangChain · Pinecone · XGBoost · LightGBM · ResNet-50 (PyTorch) ·
SageMaker · AWS Textract · Google Places API · Lovable (React/TS)
