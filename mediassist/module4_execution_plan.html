- cardiac_risk_score: age_over_50 × smoker × (hypertension OR diabetes)

  # Justification: smoking is the dominant respiratory risk factor

What NOT to do

S5
You do it
Design ResNet-50 fine-tuning strategy
notebooks/03_xray_cnn_training.ipynb
Transfer learning decisions — how many layers to freeze, what learning rates to use for frozen vs unfrozen layers, which augmentations are medically valid for chest X-rays, and what loss function handles multi-label classification. You decide all of this before Gemini writes a line of training code.
High effort ~3 hrs
Every decision you must make — write each one down with justification
DecisionYour choiceMedical/technical justification
Layers to freeze Freeze layer1-layer3 (first 6 blocks). Fine-tune layer4 + FC. Early ResNet layers learn generic edges/textures — already useful for X-rays. Only the deeper semantic layers need domain adaptation.
Learning rates New FC head: 1e-3. Fine-tuned layer4: 1e-4. Frozen layers: 0. Differential LR prevents catastrophic forgetting. 10× lower LR for pretrained weights.
Loss function Binary Cross-Entropy (BCE) per class, NOT cross-entropy. ChestX-ray14 is multi-label — a patient can have both Pneumonia AND Atelectasis. BCE treats each class independently.
Class weights in loss Positive weight per class = neg_count / pos_count for each of 14 classes. "No Finding" is 60% of data. Without weighting, model predicts "No Finding" always. Weighted BCE fixes this.
Augmentations (X-ray valid ones) Horizontal flip ✅ Random rotation ±10° ✅ Random brightness/contrast ✅ Horizontal flip is valid — left/right lung pathologies appear in both orientations. ±10° mimics patient positioning variance.
Augmentations to AVOID Vertical flip ❌ Random crop ❌ Color jitter (severe) ❌ Vertical flip creates anatomically impossible images (heart on wrong side). Cropping removes lung edges where pathology hides.
LR schedule CosineAnnealingLR over 20 epochs. Cosine annealing finds better minima than step decay for medical imaging tasks.
Evaluation metric AUC-ROC per class (not accuracy, not F1). AUC-ROC is threshold-independent and handles class imbalance. Standard metric for CheXNet benchmarks.
Interview question you must own
"Why can't you use vertical flip as a data augmentation for chest X-rays? Why did you use BCE loss instead of cross-entropy? What is differential learning rate and why did you apply it?"
