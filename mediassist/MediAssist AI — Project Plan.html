Personal Profile Layer

What personal data is collected and why

The same symptom means very different things depending on who has it. A 55-year-old male smoker with chest pain is a cardiac emergency. A 22-year-old athlete with the same symptom is probably muscle strain. Without this context, the model is guessing.

👤
Basic demographics
  • → Full name (for report personalization)
  • → Age (changes disease probability drastically)
  • → Gender (risk profiles differ significantly)
  • → Height + weight (BMI — obesity risk flag)
  • → Blood group (relevant for drug compatibility)
🏥
Medical history
  • → Existing conditions (diabetes, hypertension…)
  • → Known allergies (penicillin, NSAIDs, etc.)
  • → Current medications (drug conflict detection)
  • → Past surgeries or procedures
  • → Vaccination history (if relevant)
🚬
Lifestyle & habits
  • → Smoking (yes/no, pack-years if yes)
  • → Alcohol consumption (units per week)
  • → Physical activity level (sedentary → active)
  • → Diet type (junk / balanced / vegan)
  • → Sleep quality (hours, disruptions)
🧬
Family history
Heart disease, cancer, diabetes, and other hereditary conditions in immediate family. Significantly raises prior probability of those conditions appearing in the diagnosis. A 40-year-old with a father who had a heart attack at 45 is immediately flagged at higher cardiac risk.
📍
Location context
City and region matter medically. Dengue risk is high in Jaipur during monsoon. Malaria risk differs by zone. Air quality index affects respiratory symptom interpretation. The system uses location to apply region-specific disease priors in the RAG query.
How profile data changes the output — real example
Without personal profile
Symptom: "chest pain + shortness of breath"

Diagnosis: Possible cardiac or pulmonary issue
Urgency: Moderate
Action: "Consult a doctor soon"
With personal profile (male, 55, smoker, hypertension)
Symptom: "chest pain + shortness of breath"

Diagnosis: Possible Acute Coronary Syndrome — high cardiac risk profile
Urgency: 🚨 Emergency
Action: "Call 112 now. Stop smoking. Alert family."
How profile data flows into each system layer
XGBoost Classifier Age, gender, BMI, smoking status, and existing conditions are direct feature inputs to the symptom-to-disease model. Model accuracy improves significantly with these features vs. symptoms alone.
Urgency Model Lifestyle habits act as risk multipliers — smoker + age 50+ + chest pain automatically elevates urgency from moderate to emergency regardless of other signals.
RAG Query Patient context JSON is appended to the retrieval query — pulls age/gender-specific and condition-specific medical guidelines instead of generic articles.
LLM Prompt Full profile is injected into the system prompt. The LLM's action plan now says "You should quit smoking" instead of generic advice. Drug suggestions respect allergy and current medication fields.
Doctor Finder Filters specialists by language preference, gender preference (female gynecologist), and age specialization (pediatric for children, geriatric for 65+).
DS Analytics Stored profiles enable longitudinal analysis — track symptom progression over time, correlate lifestyle habits with conditions, power cohort-level health insights as a future feature.

Output Layer

What the system returns

Every response is a structured JSON object with three layers — diagnosis, urgency, and recommendations. The frontend renders each layer as a distinct UI panel.

🩺 Diagnosis Report
Possible condition name with confidence score, plain-language explanation, and cited source from the RAG knowledge base. Always includes a medical disclaimer.
⚡ Urgency Scoring
ML severity model classifies the case as Low, Moderate, Urgent, or Emergency. Drives the action plan — rest at home vs. book appointment vs. go to ER now.
📍 Doctor Finder
Top 3–5 specialists near the user's location, filtered by the required specialty. Returns name, rating, distance, and contact info via Google Places API / NMC Registry.
Urgency levels — what each means
● Low
Rest, hydration, monitor symptoms. Follow up with GP in 3–5 days if no improvement.
● Moderate
Book a doctor appointment within 24–48 hours. Avoid strenuous activity.
● Urgent
Visit an urgent care clinic today. Do not delay. Take prescribed medications.
🚨 Emergency
Go to the emergency room immediately or call 112. This requires immediate medical attention.

System Architecture

How the layers connect

Five logical layers, each with a clear responsibility. The RAG pipeline is the backbone; DS models run as microservices alongside it.

Layer 1 Input & Preprocessing
Receives: Text symptoms, images (X-ray / body photo), prescription scan
Does: S3 upload → Bedrock Vision → Textract OCR → spaCy NER → unified patient context JSON
Output: PatientContext { symptoms[], image_findings, ocr_text, location }
Layer 2 DS Inference Layer
Receives: PatientContext
Does: CNN image classification → Symptom classifier (XGBoost) → Severity scorer (LightGBM) → all results merged
Output: DSResults { predicted_condition, confidence, severity_level, specialty_needed }
Layer 3 RAG Pipeline
Receives: PatientContext + DSResults
Does: BioSentBERT embed query → Pinecone similarity search → retrieve top-k medical chunks → LangChain prompt → Bedrock Claude 3.5
Output: Diagnosis { condition, explanation, citations[], action_plan }
Layer 4 Doctor Finder
Receives: specialty_needed + user location (lat/lng)
Does: Query Google Places API or NMC Registry → filter by specialty + rating → rank by distance
Output: Recommendations [ { name, specialty, rating, distance, contact } ]
Layer 5 Response Assembly
Receives: All layer outputs
Does: Merge → validate → attach disclaimer → serialize to JSON → return HTTP 200
Output: Final unified JSON response to Lovable React frontend

Datasets and knowledge bases

All free / open access. These feed both the ML model training and the RAG vector knowledge base.

NIH ChestX-ray14
112,000 frontal chest X-rays with 14 disease labels. Used to train and fine-tune the ResNet-50 image classifier.
CNN Training
Kaggle: Disease Symptom Dataset
Structured mapping of symptoms to 41 common diseases. Core training data for the XGBoost symptom classifier.
ML Training
PubMed Abstracts
30M+ biomedical research abstracts. Chunked and indexed into Pinecone as the primary RAG medical knowledge base.
RAG KB
OpenFDA API
Drug labels, adverse events, recalls. Used for drug interaction checks from prescription OCR output.

SNOMED CT (subset)
